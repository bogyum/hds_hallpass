## 1. 배경 및 문제 정의

### 1.1 배경

시험기간 및 평소 수업 시간 중 학생들이 교무실에 예고 없이 출입함으로써 다음과 같은 문제가 지속적으로 발생하고 있다.

> 🔒 **핵심 문제**  
> 시험 기간 중 학생들의 무단 교무실 출입은 시험 보안에 심각한 위협이 되며, 평소에도 잦은 출입으로 인해 교사의 집중 업무 수행이 어렵다.

### 1.2 현재 문제점

- 시험 기간 중 무단 교무실 출입으로 인한 시험지·정답 보안 취약
- 수시로 방문하는 학생으로 인해 교사의 업무 흐름이 자주 끊김
- 학생 방문 기록이 남지 않아 사후 확인 불가
- 교사가 교무실 밖에서 학생을 상담·응대할 수 있는 체계 부재
- 여러 학생이 동시에 같은 교사를 찾아 대기하는 비효율 발생

### 1.3 해결 방향

교무실 입구에 태블릿을 거치하여, 학생이 앱을 통해 교사를 호출하는 방식으로 전환한다. 교사는 교무실 내부에서 요청 목록을 확인하고, 교무실 밖으로 나가 학생을 응대함으로써 보안과 업무 효율을 동시에 확보한다.

## 2. 제품 개요

### 2.1 제품명

**선생님~ 질문있어요!** (Teacher Call System)

### 2.2 제품 비전

> 🎯 교무실 출입 없이도 교사와 학생이 원활하게 소통할 수 있는 디지털 호출 플랫폼을 구축하여, 시험 보안 강화 및 교사 업무 몰입도를 향상시킨다.

### 2.3 핵심 목표 (Goals)

1. 시험 기간 교무실 무단 출입 차단 → 보안 강화
2. 교사 응대 대기 체계 구축 → 학생 편의 개선
3. 호출 이력 로그 관리 → 투명한 기록 보존
4. 학교 단위 다중 지원 → 확장성 확보
5. 태블릿/노트북 동작 → 접근성 보장

### 2.4 범위 외 (Out of Scope)

- 모바일 앱 (iOS / Android 네이티브)
- 학생 출결 관리 기능
- 교사 간 내부 메시지 기능
- 화상 상담 기능

## 3. 사용자 및 이해관계자

### 3.1 사용자 유형

| 사용자 유형 | 접근 방식 | 주요 사용 환경 | 핵심 니즈 |
|------------|----------|--------------|---------|
| 학생 | 학교코드 + 비밀번호 | 교무실 입구 태블릿 | 빠른 교사 호출, 확인 여부 확인 |
| 개별 교사 | 이름 + 학교코드 + 비밀번호 | 개인 노트북/태블릿 | 학생 호출 확인 및 처리 |
| 교무실 전체 관리 | 학교코드 + 비밀번호 | 교무실 내 공용 화면 | 전체 교사 호출 현황 한눈 파악 |
| 관리자 | 관리자 ID + PW | 학교 관리 PC | 학교코드·교사 정보 등록 및 관리 |

### 3.2 사용자 시나리오

#### 학생 시나리오

중간고사 기간, 수학 문제가 풀리지 않는 학생 김철수가 교무실 앞 태블릿 앞에 선다. 앱에 학교코드와 비밀번호를 입력하고, 수학 교사 "박지수" 버튼을 누른다. 팝업에 자신의 이름을 입력하고 호출 버튼을 클릭하면, **화면 테두리가 녹색으로 한 번 깜박이고 짧은 알림음이 울려** 호출이 정상 접수되었음을 확인한다. 이후 교무실 밖에서 대기하다 교사가 나오면 상담을 진행한다.

#### 교사 시나리오

박지수 교사는 본인 노트북에서 교사용 확인 화면에 로그인한다. 새 호출이 발생하면 **화면 상단에 배너 알림이 나타나고 알림음이 재생**되어 즉시 인지할 수 있다. "김철수" 버튼을 클릭하여 호출 시간을 확인하고, "확인" 버튼을 누른 뒤 교무실 밖으로 나가 학생을 응대한다.

## 4. 기능 요구사항

### 4.1 공통 — 진입 화면

앱 최초 접속 시 4가지 접속 모드를 선택하는 메인 화면이 표시된다.

| 버튼 라벨 | 연결 기능 | 설명 |
|----------|----------|------|
| 학생용 | 학생용 호출 화면 | 학교코드 + 공용 비밀번호로 진입 |
| 교사용 | 개별 교사 확인 화면 | 교사 이름 + 학교코드 + 개인 비밀번호로 로그인 |
| 교무실용 | 전체 교사 현황 화면 | 학교코드 + 비밀번호로 진입 |
| 관리자용 | 관리자 화면 | 관리자 ID + PW로 로그인 |

---

### 4.2 학생용 호출 화면

#### 4.2.1 인증

- 학교코드 + 공용 비밀번호 입력
- 인증 실패 시 오류 메시지 표시
- 태블릿 키오스크 특성상 자동 로그아웃 없음 (세션 유지)

#### 4.2.2 교사 선택

- 교무실별로 그룹화하여 교사 이름 버튼 표시 (5개 교무실: 1학년·2학년·3학년 교무실, 본 교무실, 특별실)
- 한 화면에서 모든 교사가 확인 가능하도록 버튼 크기를 자동 조정 (스크롤 최소화)
- 교사 버튼 클릭 시 이름 입력 팝업 표시

#### 4.2.3 호출 처리

- 팝업에서 학생 이름 입력 후 "호출" 버튼 클릭
- 동일 학생이 같은 교사에 재호출 가능 (카운트는 1개 유지, 시간 로그는 누적)
- 호출 완료 후 자동으로 초기 화면으로 복귀
- 화면 우측 상단에 현재 총 미처리 호출 수 실시간 카운트 표시

#### 4.2.4 호출 성공 피드백 ✨ NEW

- 호출 버튼 클릭 직후, 화면 전체 테두리에 **녹색 LED 효과(Green Glow Border)를 1회 깜박임** (점등 → 소등, 약 0.8초)
- 동시에 **짧은 긍정 알림음** 재생 (예: 경쾌한 "띵" 소리, 0.5초 이내)
- 이후 "호출이 접수되었습니다" 토스트 메시지를 0.8초간 표시하고 초기 화면으로 복귀

> 📌 **UX 포인트**  
> 시각 + 청각 피드백을 동시에 제공하여, 화면을 보지 않고 있는 학생도 호출 성공 여부를 즉시 인지할 수 있도록 한다. 알림음은 교실 소음 환경을 고려하여 볼륨을 적절히 설정(기본값 70%)하며, 관리자 화면에서 조절 가능하다.

#### 4.2.5 확인 알림

- 교사가 확인 버튼 클릭 시, 학생용 화면 하단에 "OOO 선생님 확인하셨습니다" 1초간 표시

---

### 4.3 개별 교사용 확인 화면

#### 4.3.1 인증

- 교사 이름 선택 또는 입력 + 학교코드 + 개인 비밀번호로 로그인
- 본인 호출 데이터만 조회 가능

#### 4.3.2 호출 목록 표시

- 자신을 호출한 학생 버튼 목록 표시
- 각 버튼 좌측 상단에 녹색 점 깜박임 애니메이션으로 미확인 상태 표시
- 버튼 클릭 시 하단에 해당 학생의 호출 시간 목록 표시 (최신순)

#### 4.3.3 신규 호출 알림 ✨ NEW

- Firestore onSnapshot으로 새 호출 발생을 실시간 감지
- **시각 알림:** 화면 상단에 슬라이드인 배너 알림 표시 ("OOO 학생이 호출했습니다", 3초 후 자동 소멸)
- **청각 알림:** 새 호출 발생 시 알림음 재생 (교사 업무 환경에 적합한 부드러운 알림음, 예: 차임벨 2회)
- 알림음은 브라우저 자동재생 정책(Autoplay Policy)을 준수하여, **교사가 최초 1회 화면을 클릭(인터랙션)한 후부터 활성화**
- 교사 화면 내 **알림음 ON/OFF 토글 버튼** 제공 (기본값: ON)

#### 4.3.4 확인 처리

- "확인" 버튼 클릭 시 해당 학생의 모든 미확인 호출 기록을 일괄 확인 처리
- 확인 처리 후 해당 버튼 제거 (또는 비활성화 처리)
- 처리된 건수만큼 학생용 화면 카운터 감소
- 확인 교사 이름이 학생 화면에 1초간 표시

#### 4.3.5 실시간 업데이트

- Firebase Realtime Database 구독으로 새 호출 발생 시 즉시 목록 갱신

---

### 4.4 전체 교사용 확인 화면 (교무실용)

#### 4.4.1 인증

- 학교코드 + 교무실용 비밀번호로 진입

#### 4.4.2 교사 현황 표시

- 교무실별 (5개) 섹션으로 구분하여 전체 교사 버튼 표시
- 호출이 1건 이상 있는 교사: **녹색 버튼**
- 호출이 없는 교사: **회색 버튼**
- 교사가 확인 처리 완료 시 해당 버튼이 녹색 → 회색으로 실시간 전환

#### 4.4.3 실시간 업데이트

- Firebase 구독으로 버튼 상태 실시간 동기화 (폴링 없이 WebSocket 기반)

---

### 4.5 관리자 화면

#### 4.5.1 학교코드 관리

- 새 학교코드 등록 / 수정 / 삭제
- 학교코드별 학생용·교무실용 공용 비밀번호 설정

#### 4.5.2 교사 관리

- 교사 이름, 소속 교무실, 개인 비밀번호 등록 / 수정 / 삭제
- 학교코드별 독립 관리 (타 학교 데이터 열람 불가)

#### 4.5.3 알림 설정 관리 ✨ NEW

- 학생용 호출 성공 알림음 볼륨 설정 (0~100%, 기본값 70%)
- 알림음 종류 선택 (사전 제공 3종: 기본 띵 / 차임벨 / 단순 비프)
- 설정 저장 시 해당 학교코드의 모든 화면에 즉시 반영

#### 4.5.4 로그 조회

- 날짜, 교사, 학생 이름 기준 호출 로그 검색 및 내보내기 (CSV)

## 5. 알림 및 피드백 시스템 (NEW v1.1)

### 5.1 개요

| 화면 | 트리거 | 시각 효과 | 청각 효과 |
|------|--------|----------|----------|
| **학생용** | 호출 버튼 클릭 성공 | 화면 테두리 녹색 LED 1회 깜박임 (0.8초) | 짧은 긍정음 1회 (0.5초) |
| **교사용** | 새 호출 수신 | 상단 슬라이드인 배너 (3초) + 학생 버튼 녹색 점 애니메이션 | 차임벨 알림음 2회 |

---

### 5.2 학생용 — 녹색 LED 테두리 효과

#### 동작 명세

```
1. 호출 Firestore 저장 성공 응답 수신
2. 화면 전체를 덮는 fixed 오버레이(border-only) 레이어 활성화
3. 0~200ms: 녹색 border + box-shadow glow 점등 (opacity 0 → 1)
4. 200~600ms: 유지
5. 600~800ms: 소등 (opacity 1 → 0)
6. 레이어 비활성화 → 토스트 메시지 표시 → 초기 화면 복귀
```

#### 구현 가이드

```css
/* 예시: Tailwind + CSS custom */
.led-overlay {
  position: fixed;
  inset: 0;
  pointer-events: none;
  border: 6px solid #22c55e;          /* green-500 */
  box-shadow: 0 0 24px 6px #22c55e,
              inset 0 0 24px 4px #22c55e80;
  border-radius: 0;
  animation: ledBlink 0.8s ease-out forwards;
  z-index: 9999;
}

@keyframes ledBlink {
  0%   { opacity: 0; }
  25%  { opacity: 1; }
  75%  { opacity: 1; }
  100% { opacity: 0; }
}
```

#### 유의사항

- Firestore 저장 **성공** 콜백 이후에만 효과 실행 (저장 실패 시 미실행, 오류 메시지 표시)
- 효과 재생 중 중복 호출 방지 (버튼 일시 비활성화)

---

### 5.3 학생용 — 호출 성공 알림음

#### 동작 명세

- Web Audio API 또는 `<audio>` 태그로 구현
- 재생 시점: LED 효과 시작과 동시
- 기본 음원: 관리자가 선택한 음원 파일 (Firebase Storage 또는 번들 내 정적 파일)
- 재생 길이: 0.5초 이내
- 볼륨: 관리자 설정값 반영 (기본 70%)

#### 브라우저 자동재생 정책 대응

태블릿(크롬 기반)에서는 사용자 인터랙션 이전 오디오 자동재생이 차단될 수 있다.

```
대응 방법:
- 학교코드 로그인 완료 시점(버튼 클릭)을 인터랙션 기준으로 사용
- 로그인 직후 AudioContext를 resume() 상태로 초기화
- 이후 모든 알림음은 정상 재생됨
```

### 5.4 교사용 — 신규 호출 알림 배너

#### 동작 명세

```
1. Firestore onSnapshot → 신규 calls 문서 감지 (calledAt이 현재 세션 시작 이후인 문서)
2. 화면 상단에서 배너 슬라이드인 (translateY: -100% → 0, 0.3초)
3. 배너 내용: "📣 [학생이름] 학생이 호출했습니다 — [HH:MM:SS]"
4. 3초 유지 후 슬라이드아웃 (0.3초)
5. 복수 호출 시 큐(Queue) 방식으로 순차 표시 (최대 3개 누적, 이후 "외 N건" 처리)
```

### 5.5 교사용 — 알림음

#### 동작 명세

- 신규 호출 감지 시 알림음 재생
- 권장 음원: 차임벨 2회 또는 부드러운 알림음 (업무 환경 고려, 0.8~1.5초)
- 교사 화면 우측 상단 **🔔 ON / 🔕 OFF 토글** 제공
- 토글 상태는 localStorage에 저장 (세션 유지)

#### 브라우저 자동재생 정책 대응

- 로그인 버튼 클릭 시점을 인터랙션 기준으로 사용
- 로그인 완료 직후 AudioContext.resume() 호출
- 이후 모든 알림음 정상 재생 보장

#### 복수 호출 알림음 처리

- 3초 이내 다중 호출 발생 시 알림음 1회만 재생 (debounce 처리)
- 단, 배너는 각 호출별로 개별 표시

### 5.6 음원 파일 명세

| 코드명 | 용도 | 길이 | 파일 형식 | 비고 |
|--------|------|------|----------|------|
| `sound_call_success` | 학생 호출 성공 | ≤ 0.5초 | mp3, ogg | 경쾌하고 짧은 긍정음 |
| `sound_teacher_notify` | 교사 신규 호출 알림 | ≤ 1.5초 | mp3, ogg | 차임벨 계열, 부드러운 음색 |

- 음원 파일은 Next.js `/public/sounds/` 에 번들로 포함
- 관리자 화면에서 교체 가능 (Firebase Storage 업로드 후 URL 저장 방식)
- 브라우저 호환성을 위해 mp3 + ogg 두 포맷 동시 제공

## 7. 비기능 요구사항

### 7.1 성능

- 호출 접수 후 교사 화면 반영: **2초 이내**
- 확인 처리 후 학생 화면 반영: **2초 이내**
- 교무실용 현황 화면 버튼 상태 변경: **3초 이내**
- LED 효과 + 알림음 시작: 호출 저장 성공 후 **100ms 이내**
- 교사 알림 배너 표시: 호출 발생 후 **2초 이내**
- 최대 동시 사용자: 학교당 100명 이상

### 7.2 보안

- 모든 비밀번호 Firebase Authentication 또는 서버 측 해시(bcrypt) 처리
- 학교코드 기반 데이터 완전 분리 (Firestore Security Rules 적용)
- HTTPS 필수 적용

### 7.3 가용성 및 유지보수

- Firebase 기반 서버리스 운영 → 별도 서버 관리 불필요
- Next.js Vercel 또는 Firebase Hosting 배포
- 학교별 데이터 격리로 한 학교 장애가 타 학교에 영향 없음

### 7.4 호환성

- 크롬(Chrome) 최신 버전 기준 최적화
- 태블릿 (10인치 이상) 및 노트북 (13인치 이상) 반응형 레이아웃
- 터치 인터페이스 및 마우스/키보드 모두 지원
- Web Audio API: Chrome 66+, Safari 14.1+, Firefox 76+ 지원 확인 필요

### 7.5 디자인 요구사항

- 밝고 깨끗한 Light Mode UI (다크모드 미지원)
- Primary 컬러: Blue (#2563EB) 계열
- 호출 상태 색상: 녹색(호출 있음) / 회색(호출 없음) 명확한 구분
- 학생용 호출 화면: 한 화면에 모든 교사 이름 버튼 표시 (스크롤 최소화)
- 최신 디자인 트렌드 (Rounded corners, shadow, clean typography)
- 교무실별 섹션 구분을 위한 Card/Group UI 적용

## 9. 화면 흐름 (User Flow)

### 9.1 학생 호출 흐름

1. 앱 접속 → 메인 화면 (학생용 / 교사용 / 교무실용 / 관리자용 선택)
2. "학생용" 선택 → 학교코드 + 비밀번호 입력
3. 인증 성공 → **AudioContext.resume() 초기화 (알림음 활성화 준비)**
4. 교무실별 교사 목록 화면 진입
5. 교사 버튼 클릭 → 이름 입력 팝업 표시
6. 이름 입력 후 "호출" 클릭 → Firestore 저장
7. 저장 성공 → **녹색 LED 테두리 깜박임 + 알림음 동시 실행 (0.8초)**
8. 토스트 메시지 표시 (0.8초) → 초기 화면 복귀

### 9.2 교사 확인 흐름

1. "교사용" 선택 → 이름 + 학교코드 + 비밀번호 로그인
2. 로그인 완료 → **AudioContext.resume() 초기화** + Firestore 구독 시작
3. 본인 호출 목록 표시 (실시간 구독)
4. 신규 호출 발생 → **상단 배너 슬라이드인 + 알림음 재생**
5. 학생 버튼 클릭 → 호출 시간 목록 표시
6. "확인" 클릭 → confirmedAt 업데이트, 학생 화면에 알림 표시
7. 교사 교무실 밖으로 나가 학생 응대

### 9.3 교무실 현황 흐름

1. "교무실용" 선택 → 학교코드 + 비밀번호 입력
2. 전체 교사 현황 화면 표시 (교무실별 그룹)
3. 호출 발생/확인 시 버튼 색상 실시간 자동 변경 (녹색 ↔ 회색)

## 10. Firebase Security Rules (핵심 원칙)

> 🔐 **보안 원칙**  
> 모든 Firestore 읽기/쓰기는 요청자의 schoolCode와 데이터의 schoolCode가 일치하는 경우에만 허용한다. 관리자는 자신에게 할당된 schoolCode만 관리 가능하다.

### 10.1 규칙 적용 방향

- `schools`: 인증된 학교코드 소유자만 읽기 가능, 쓰기는 관리자만
- `teachers`: 동일 학교코드 사용자만 읽기, 쓰기는 관리자
- `calls`: 동일 학교코드 사용자 읽기/쓰기 허용, 단 `confirmedAt` 업데이트는 교사 계정만
- `admins`: 관리자 본인 계정만 읽기, 쓰기 불가 (관리자 콘솔에서만)
